Repository: `https://github.com/christopherklint97/homebrew-tap`

## Project Structure

```
homebrew-tap/
├── clockr.rb    # Formula for clockr (AI-powered Clockify CLI)
├── README.md    # User-facing documentation
└── CLAUDE.md    # This file
```

## How Homebrew Taps Work

- The repo must be named `homebrew-tap` on GitHub.
- Users add it with `brew tap christopherklint97/tap`.
- Homebrew discovers all `.rb` formula files in the repo root.

## Formulae

Formula files are Ruby classes that extend `Homebrew::Formula`. Each formula specifies:
- `desc`, `homepage`, `version`, `license` metadata
- Platform/architecture-specific download URLs and SHA256 checksums (`on_macos`, `on_linux` blocks)
- Install instructions (typically `bin.install "<binary>"`)

## Adding a New Formula

Formulae in this tap are auto-generated by **GoReleaser** during the release process of each tool. To add a new formula:

1. In the source project's `.goreleaser.yml`, configure the `brews` section to publish to this tap repo.
2. When a new release is tagged and GoReleaser runs, it will create or update the `.rb` file in this repo automatically.
3. Do not manually edit GoReleaser-generated formula files -- they will be overwritten on the next release.

## Conventions

- **Do not manually edit formula files** -- they are generated by GoReleaser (noted in file header comments).
- Formula files live at the repo root (not in subdirectories).
- Commit messages follow the pattern: `Brew formula update for <tool> version <version>`.
- Default branch: `main`.
